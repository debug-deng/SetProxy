Rust 配合 Tauri 能够非常轻松地打包成单文件、绿色版（Portable）exe。

默认情况下，Tauri 的构建流程会生成一个 .msi 安装包，但你可以通过简单的配置，让它输出一个独立的 .exe 文件，用户双击即可运行，无需安装过程。

以下是实现单文件运行的技术细节和打包步骤：

1. 如何配置打包为单文件 exe
在 Tauri 项目的配置文件 src-tauri/tauri.conf.json 中，你需要调整 bundle 选项。

JSON
{
  "tauri": {
    "bundle": {
      "active": true,
      "targets": ["nsis"], // 使用 NSIS 制作安装程序或单文件
      "identifier": "com.proxy.manager",
      "icon": ["icons/32x32.png", "icons/128x128.png"]
    }
  }
}
关键点：
虽然 nsis 通常用于生成安装程序，但你可以通过 CLI 参数直接提取出编译好的二进制文件。在运行打包命令时：

Bash
cargo tauri build
编译完成后，你可以在以下目录找到你的独立 exe 文件：
src-tauri/target/release/your-app-name.exe

这个 exe 就是编译后的完整二进制文件。它已经包含了：

Rust 后端逻辑：修改注册表、系统 API 调用。

前端 UI 资源：HTML/JS/CSS 已经被静态嵌入到二进制文件中。

Tauri 运行时：负责桥接前端和后端。

2. 为什么 Rust 实现的单文件非常可靠？
真正的原生代码：不同于 Python（需要打包一个庞大的解释器环境）或 Java（需要虚拟机），Rust 编译出的是机器码。

极致轻量：因为使用了 Windows 系统自带的 WebView2 控件来渲染 UI，你的 exe 文件体积通常只有 3MB - 8MB 左右。

无依赖运行：除了现代 Windows 系统自带的 Edge/WebView2 运行时（现在 Win10/11 都是标配），不需要用户安装任何额外的库或框架。

3. 实现“绿色版”运行的注意事项
为了保证程序在“非安装”模式下也能完美运行，在代码实现逻辑上需要注意以下几点：

A. 配置文件的存放位置
既然是绿色版，你可能不希望在系统里留下太多痕迹。你可以选择：

便携模式：将 JSON 配置文件存放在 exe 同级目录下。

标准模式：使用 tauri::api::path::app_data_dir() 存放在用户的 AppData 目录下。

推荐做法（Rust 逻辑）：

Rust
// 获取 exe 所在的当前路径，实现真正的绿色软件
let mut config_path = std::env::current_exe().unwrap();
config_path.pop();
config_path.push("config.json");
B. 注册表操作的安全性
正如之前提到的，修改 HKEY_CURRENT_USER 不需要管理员权限。这意味着你的单文件 exe 不需要“以管理员身份运行” 就能实现代理切换功能，这极大地降低了用户的信任门槛。

1. 后端核心源码 (src-tauri/src/main.rs)
这段代码实现了代理的激活、恢复以及“免重启”刷新逻辑。

Rust
#![cfg_attr(not(debug_assertions), windows_subsystem = "windows")]

use serde::{Deserialize, Serialize};
use std::ptr;
use winreg::enums::*;
use winreg::RegKey;
// 导入 Windows 系统 API 用于强制刷新网络设置
use windows_sys::Win32::Networking::WinInet::{
    InternetSetOptionW, INTERNET_OPTION_REFRESH, INTERNET_OPTION_SETTINGS_CHANGED,
};

#[derive(Serialize, Deserialize)]
struct CommandResult {
    success: bool,
    message: String,
}

/// 核心逻辑：通知 Windows 系统代理设置已更改，防止浏览器缓存旧设置
fn flush_system_proxy_settings() {
    unsafe {
        // 通知设置已更改
        InternetSetOptionW(ptr::null_mut(), INTERNET_OPTION_SETTINGS_CHANGED, ptr::null_mut(), 0);
        // 强制刷新设置
        InternetSetOptionW(ptr::null_mut(), INTERNET_OPTION_REFRESH, ptr::null_mut(), 0);
    }
}

/// 命令：激活系统代理
#[tauri::command]
fn enable_proxy(ip: String, port: u32) -> CommandResult {
    let hkcu = RegKey::predef(HKEY_CURRENT_USER);
    let path = "Software\\Microsoft\\Windows\\CurrentVersion\\Internet Settings";
    
    match hkcu.open_subkey_with_flags(path, KEY_SET_VALUE) {
        Ok(key) => {
            let proxy_server = format!("{}:{}", ip, port);
            // 写入注册表：1 代表开启代理
            if let Err(e) = key.set_value("ProxyEnable", &1u32) {
                return CommandResult { success: false, message: e.to_string() };
            }
            // 写入代理地址和端口
            if let Err(e) = key.set_value("ProxyServer", &proxy_server) {
                return CommandResult { success: false, message: e.to_string() };
            }
            
            flush_system_proxy_settings();
            CommandResult { success: true, message: "系统代理已成功激活".into() }
        }
        Err(e) => CommandResult { success: false, message: format!("无法打开注册表: {}", e) },
    }
}

/// 命令：恢复（关闭）系统代理
#[tauri::command]
fn restore_proxy() -> CommandResult {
    let hkcu = RegKey::predef(HKEY_CURRENT_USER);
    let path = "Software\\Microsoft\\Windows\\CurrentVersion\\Internet Settings";

    match hkcu.open_subkey_with_flags(path, KEY_SET_VALUE) {
        Ok(key) => {
            // 写入注册表：0 代表关闭代理
            if let Err(e) = key.set_value("ProxyEnable", &0u32) {
                return CommandResult { success: false, message: e.to_string() };
            }
            
            flush_system_proxy_settings();
            CommandResult { success: true, message: "系统代理已恢复初始状态".into() }
        }
        Err(e) => CommandResult { success: false, message: format!("操作失败: {}", e) },
    }
}

fn main() {
    tauri::Builder::default()
        .invoke_handler(tauri::generate_handler![enable_proxy, restore_proxy])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}

4. 方案逻辑总结
GUI 实现：用户在界面点击“激活”。

指令下发：前端通过 invoke('enable_proxy', { ip, port }) 发送请求。

注册表修改：Rust 后端接收指令，静默修改 HKCU 注册表，无需管理员权限弹窗。

强制生效：通过 InternetSetOptionW 接口捅一下 Windows 的网络驱动，让更改立刻全局生效。

退出/恢复：用户点击“恢复系统代理”，注册表重置为 0，系统瞬间回到直连状态。
