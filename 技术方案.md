技术方案：基于 Rust 的系统代理管理工具
1. 项目架构概述
该方案采用 Tauri 框架。Tauri 的核心逻辑是：前端 (Webview) 负责 UI 交互与配置展示，后端 (Rust) 负责高权限的系统操作（注册表修改、文件读写）。

架构图示
Frontend: Vue 3 / React (管理配置列表、发送切换指令)

IPC (进程间通信): Tauri Commands

Backend (Rust): 执行注册表变更、调用 Windows API 刷新系统设置、JSON 持久化

3. 核心业务逻辑实现 (Rust)
3.1 修改 Windows 注册表
操作 Windows 系统代理，本质上是修改 HKEY_CURRENT_USER 下的特定键值。

依赖项 (Cargo.toml):

Ini, TOML
[dependencies]
tauri = { version = "1.5", features = ["shell-open"] }
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
winreg = "0.50" # 用于操作注册表
windows-sys = { version = "0.48", features = ["Win32_Networking_WinInet"] } # 用于刷新系统设置
实现代码 (src-tauri/src/proxy_manager.rs):

Rust
use winreg::enums::*;
use winreg::RegKey;
use windows_sys::Win32::Networking::WinInet::{
    InternetSetOptionW, INTERNET_OPTION_SETTINGS_CHANGED, INTERNET_OPTION_REFRESH,
};

pub struct ProxyManager;

impl ProxyManager {
    // 激活指定代理
    pub fn apply_proxy(ip: &str, port: u32) -> Result<(), String> {
        let hkcu = RegKey::predef(HKEY_CURRENT_USER);
        let path = "Software\\Microsoft\\Windows\\CurrentVersion\\Internet Settings";
        let key = hkcu.open_subkey_with_flags(path, KEY_SET_VALUE).map_err(|e| e.to_string())?;

        let proxy_server = format!("{}:{}", ip, port);
        
        key.set_value("ProxyEnable", &1u32).map_err(|e| e.to_string())?;
        key.set_value("ProxyServer", &proxy_server).map_err(|e| e.to_string())?;

        Self::flush_settings();
        Ok(())
    }

    // 恢复系统代理（关闭）
    pub fn disable_proxy() -> Result<(), String> {
        let hkcu = RegKey::predef(HKEY_CURRENT_USER);
        let path = "Software\\Microsoft\\Windows\\CurrentVersion\\Internet Settings";
        let key = hkcu.open_subkey_with_flags(path, KEY_SET_VALUE).map_err(|e| e.to_string())?;

        key.set_value("ProxyEnable", &0u32).map_err(|e| e.to_string())?;

        Self::flush_settings();
        Ok(())
    }

    // 核心逻辑：通知 Windows 系统设置已更改（否则浏览器可能不会立即生效）
    fn flush_settings() {
        unsafe {
            InternetSetOptionW(std::ptr::null_mut(), INTERNET_OPTION_SETTINGS_CHANGED, std::ptr::null_mut(), 0);
            InternetSetOptionW(std::ptr::null_mut(), INTERNET_OPTION_REFRESH, std::ptr::null_mut(), 0);
        }
    }
}
4. 前后端交互逻辑 (Tauri Commands)
在 main.rs 中暴露接口给前端调用：

Rust
#[tauri::command]
fn cmd_enable_proxy(ip: String, port: u32) -> CommandResult {
    match ProxyManager::apply_proxy(&ip, port) {
        Ok(_) => CommandResult { success: true, message: "代理已激活".into() },
        Err(e) => CommandResult { success: false, message: e },
    }
}

#[tauri::command]
fn cmd_restore_proxy() -> CommandResult {
    match ProxyManager::disable_proxy() {
        Ok(_) => CommandResult { success: true, message: "系统代理已恢复".into() },
        Err(e) => CommandResult { success: false, message: e },
    }
}

6. 实现细节补遗
数据持久化：

推荐使用 tauri-plugin-store 插件，或者在 Rust 端使用 std::fs 直接将 Vec<ProxyConfig> 序列化为 JSON 存储在 AppData 目录下。

权限处理：

修改 HKEY_CURRENT_USER 下的 Internet Settings 通常不需要管理员权限，这对用户体验非常友好。

状态回显：

程序启动时，Rust 端应读取一次注册表的 ProxyEnable 和 ProxyServer 值，反馈给前端，以显示当前系统实际运行的是哪个代理。

7. 方案优势总结
可靠性：使用 Rust 的 winreg 和 windows-sys 直接与系统 API 通信，避开了脚本调用的不确定性。

低占用：相比 Electron 动辄几百 MB 的内存，此方案运行内存通常在 20-50MB 左右。

交互优良：通过 InternetSetOptionW 实现了“免重启、即时生效”的切换体验。
